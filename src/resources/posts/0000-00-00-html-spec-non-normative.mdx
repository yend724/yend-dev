[HTML Living Standard](https://html.spec.whatwg.org/) を読んでいると、しばしば `This section is non-normative` という記述を目にする。あまり深く考えたことがなかったので、この `non-normative` という用語について調べてみた。

## HTML Living Standardでの使用例

HTML Living Standard では、次のような記述を見かける。

> This section is non-normative.

日本語訳：

> このセクションは非規範的です。

## `normative` と `non-normative` の定義

自分が見た限り HTML Living Standard 自体には `non-normative` (ないしは `normative`) の明示的な定義は記載されていないようだった。これは、W3CやIETFなどの技術仕様書で一般的に使われる標準的な用語らしい。

### W3C文書での定義

たとえばW3Cの古い文書であるUser Agent Accessibility Guidelines 1.0（2001年）には、明確な定義がある：

> **Normative（規範的）**：「その文書の要件が最も正確に述べられるために依存するもの」を指す。規範的なものは適合性のために必要。
>
> **Informative/Non-normative（参考的/非規範的）**：適合性のために決して必要とされない。

## 結論

「non-normative」は、仕様書において「参考情報」であることを示すマーカー。実装者にとっては、以下のような判断基準になる：

- **Normative**：必ず守らなければならない仕様（MUST/SHALL/SHOULD）
- **Non-normative**：理解を深めるための補足情報
