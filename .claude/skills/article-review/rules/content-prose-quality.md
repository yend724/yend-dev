## 日本語として正しく適切で、冗長な表現がない

文法・語彙・表記が正しく、文章が冗長でないことを確認する。書き終えた直後は気づきにくいが、通読すると違和感のある語順、誤った助詞、無駄な修飾が見つかる。

### 日本語として正しい

**Bad(助詞の誤り / 主述のねじれ):**

```md
このオプションがを有効にすると、キャッシュを使うことになる。
```

```md
本記事の目的は、〇〇を実装し、△△を理解することについて述べる。
```

→ 後者は「目的は」と「述べる」が呼応しておらず、主述がねじれている。

**Good:**

```md
このオプションを有効にすると、キャッシュが使われる。
```

```md
本記事では、〇〇を実装し、△△を理解することを目的とする。
```

**チェック観点:**

- 助詞(が / を / に / で / へ / と)の選択が適切か
- 主語と述語が**呼応**しているか(主述のねじれ)
- 動詞の活用形が正しいか(「立ち上げれる」のような可能動詞の誤りなど)
- 接続詞(「しかし」「したがって」「つまり」)が**前後の論理関係**と合っているか
- 句読点(「、」「。」)の打ち方が読点過多/過少になっていないか
- カタカナ語の表記が一般的か(マネージャー / マネージャ など)

### 適切な表現

**チェック観点:**

- 砕けすぎ / 硬すぎる語が記事のトーンに合っているか
- 「〜してもよい」「〜したほうがよい」など、断言の度合いが文意に合っているか
- 翻訳調の不自然な言い回しが残っていないか(「〜することができます」を多用、長すぎる修飾句、無生物主語の不自然な使用)
- 比喩・俗語・流行語が読者層に対して機能するか

### 冗長な表現を削る

**Bad(冗長 / 重複):**

```md
この関数は、引数として与えられる値を受け取って、その値を 2 倍にした値を返す関数である。
```

→ 「値」「関数」が繰り返され、長い割に情報量が少ない。

**Good:**

```md
この関数は引数を 2 倍にして返す。
```

**チェック観点:**

- **定型的な冗長表現**を圧縮できないか(「〜することができる」 → 「〜できる」、「〜することにより」 → 「〜することで」、「〜という事実」 → 「〜こと」など)
- **同じ語が 1 文中で繰り返されていないか**(短い言い換えで済むケースは多い)
- **修飾句が長すぎて係り受けが追いにくくないか**(長い修飾は分割して別の文にする)
- **形式的な前置き**(「〜ですが、〜」「〜の通り、〜」)が情報を持たない場合は削れないか
- **同じ内容を別表現で 2 度説明していないか**(整合性ではなく冗長性の問題)
- **強調の多用**(「とても」「非常に」「すごく」)で文の輪郭がぼやけていないか

**確認手段:**

- 各段落を音読してリズムが詰まる箇所・息が続かない箇所を見つける
- 「この一文がなくても意味が通るか?」を文単位で問う
- 校正ツールや AI に「自然な日本語に書き直して」と頼んで提案を比較する
