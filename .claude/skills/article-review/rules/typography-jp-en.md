## 日本語と英語の間に半角スペースを入れない

日本語の本文中に英単語、inline code、URL、markdown 装飾が現れるとき、その前後に半角スペースを入れない。レンダリング時の視認性はフォント側で確保されるため、本文側で空白を入れる必要はない。

**Bad(スペース入り):**

```md
このプロジェクトは React で書かれている。

詳細は [公式ドキュメント](https://example.com/) を参照のこと。

これは **重要** な箇所である。
```

**Good:**

```md
このプロジェクトはReactで書かれている。

詳細は[公式ドキュメント](https://example.com/)を参照のこと。

これは**重要**な箇所である。
```

**対象になる境界:**

- 日本語 ↔ 英単語(`TypeScript` 等)
- 日本語 ↔ inline code(`` `never` `` 等)
- 日本語 ↔ markdown link(`[text](url)`)
- 日本語 ↔ bold / italic(`**text**`、`*text*`)

**例外(スペースを保つ):**

- コードブロック内(``` ``` で囲まれた領域)
- 純英語の引用ブロック内
- markdown テーブルでセル境界の整列に必要なパディングスペース(`| 集合演算 |` の左右)
- リスト記法の先頭(`- 項目` の `- ` 直後)

**チェック観点:**

- ASCII printable 1 文字以上 + 半角スペース + 日本語 のパターンが本文に残っていないか
- 日本語 + 半角スペース + ASCII printable 1 文字以上 のパターンも同様
